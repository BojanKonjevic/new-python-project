Architecture

How it works

Three layers that produce a complete project with no runtime dependency on Zenit itself.

📐

Templates

A base project structure — directories, core files, dependencies. Choose blank or fastapi.

🧩

Addons

Incremental additions layered on top. Add or remove at any time without breaking your project.

⚡

Your project

A complete, ready-to-run codebase. No runtime dependency on Zenit. Ever.

Operations

Stay in control

Verify, manage, and clean up your project at any time with four commands.

zenit add <addon>

Installs dependencies, creates files, injects code, and updates configuration. Reversible at any time.

zenit remove <addon>

Deletes files, dependencies, compose services, env vars, and code injections. Clean and complete.

zenit doctor

Verifies every file, injection, and dependency against the manifest. Catches drift before it becomes a problem.

zenit list

See all available templates and addons. Check what's installed and what you can add next.

Trust

Built to be transparent

No regex hacks, no hidden state. Zenit tracks exactly what it touched and can undo all of it.

🔗

Structural injection

Code inserted via libcst — AST-aware, formatter-resilient. No regex hacks that break on style changes.

🔒

Fingerprinted manifest

Every injected block is hashed. Zenit knows exactly what it wrote and can verify or remove it safely.

🔓

Zero lock-in

Delete .zenit.toml and uv remove zenit. Your project runs without us.
